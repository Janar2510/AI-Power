# Contributing

## Workflow

1. Create a branch from `main`
2. Make changes; one feature/fix per PR
3. Run tests before submitting
4. Request review

## Code Style

- Python: PEP 8; type hints where practical
- TypeScript/JS: Consistent formatting
- No large refactors without approval

## Tests Required

- New features must include tests
- Parity changes must not break existing tests
- See [docs/test_plan.md](docs/test_plan.md)

## Security

- Report vulnerabilities privately
- Never commit secrets or API keys

## Review Gates

- CI passes
- At least one approval
- No merge conflicts
